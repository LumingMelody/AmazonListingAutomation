## 方案概述

本方案基于现有 BU2Ama Excel 颜色加色系统，采用"平台化升级"策略，而非推倒重来。当前系统已具备：
- Excel 处理引擎（加色加码）
- SKC 跟卖匹配引擎
- 导出历史管理
- 异步任务处理

**核心策略**: 保留引擎能力，外包编排层，分阶段实现 8 个运营模块的自动化支持。

### 1.2 核心组件说明

#### 接入层
- **Excel 上传**: 复用现有上传接口
- **Amazon 报表**: All Listings、Business Reports
- **SP-API**: Listings、Feeds、Reports、Orders
- **广告数据**: Amazon Ads API 或报表导入
- **售后数据**: 退款、拒付、客诉数据

#### 领域服务层
- **Catalog Service**: 上新整体规范、店铺准入标准
- **Content Service**: 五点描述生成、尺码话术模板
- **Variant Service**: 加色加码策略与库存联动
- **FollowSell Service**: SKC→老款映射→尺码拉齐（复用现有引擎）
- **Compliance Service**: 商标/IP词/禁用词扫描
- **Experiment Service**: 测试款分层与晋级决策

#### 规则与决策层
- **规则引擎**: 硬闸门（侵权、拒付率）、软评分（转化、毛利）
- **阈值中心**: 按类目/季节动态阈值，支持版本化
- **决策流**: 淘汰/观察/放量自动判定

#### 数据与分析层
- **OLTP**: PostgreSQL（任务、规则、主数据）
- **分析层**: ClickHouse/BigQuery（广告-订单-售后归因）
- **缓存队列**: Redis + Celery/RQ（异步任务、预警）

#### 编排与通知层
- **工作流**: Airflow/Prefect（日报、回流、分层任务）
- **通知**: 企业微信/飞书/Slack（阈值预警、人工审批）

### 5.4 处理器收敛

**当前问题**: 存在两个跟卖处理器
- `followsell_processor.py`: Excel 批量替换
- `follow_sell_processor.py`: SKC+新老款映射+EP索引

**解决方案**: 合并为单一 `FollowSellDomainService`

```python
# backend/app/core/followsell_domain_service.py
class FollowSellDomainService:
    """统一的跟卖领域服务"""

    def __init__(self):
        self.skc_matcher = SKCMatcher()  # 从 follow_sell_processor
        self.excel_builder = ExcelBuilder()  # 从 followsell_processor
        self.compliance_checker = ComplianceChecker()

    def process_followsell(
        self,
        old_file: Path,
        new_product_code: str,
        options: Dict[str, Any]
    ) -> JobResult:
        # 1. SKC 匹配
        skc_result = self.skc_matcher.match(old_file, new_product_code)

        # 2. 风险预检
        risk_result = self.compliance_checker.check(skc_result)

        # 3. Excel 生成
        if risk_result.is_safe:
            output = self.excel_builder.build(skc_result, options)

        # 4. 质检
        qa_score = self._quality_check(output)

        return JobResult(
            output_file=output,
            qa_score=qa_score,
            risk_result=risk_result,
            next_action="approve" if qa_score > 0.8 else "review"
        )
```

### 5.5 API 路由升级

#### 新增统一工作流入口
```python
# backend/app/api/workflow.py
@router.post("/api/workflow/run")
async def run_workflow(request: JobRequest):
    """统一工作流入口"""

    # 1. 路由到对应服务
    if request.job_type == "follow-sell":
        service = followsell_domain_service
    elif request.job_type in ["add-color", "add-code"]:
        service = variant_domain_service
    else:
        service = listing_domain_service

    # 2. 执行处理
    result = await service.process(request)

    # 3. 记录历史
    export_history.add_record(
        module=request.job_type,
        template=request.template_type,
        params=request.dict(),
        result=result.dict()
    )

    return result
```

#### 保留现有接口（向后兼容）
```python
# backend/app/api/excel.py
@router.post("/api/process")
async def process_excel(request: ProcessRequest):
    """现有接口，内部转发到工作流"""
    job_request = JobRequest(
        job_type="add-color" if request.mode == "add-color" else "add-code",
        template_type=request.template_type,
        generated_skus=request.skus,
        source_style_map={},
        risk_flags=[],
        metadata=request.dict()
    )
    return await run_workflow(job_request)
```
